# ADR-021: v23.6.0 Site Accuracy Sync

## Status: Accepted

## Context

The VoidForge marketing site (v2.8.0) is two methodology versions behind the upstream (v23.5.3 → v23.6.0). A 34-finding site audit identified content inaccuracies, missing features, stale counts, and per-universe agent discrepancies. Additionally, speech bubble entrance animations have been deferred across two sessions.

### Key upstream changes (v23.5.4 + v23.6.0):
- **Silver Surfer** (Norrin Radd) added as agent #264 — Haiku scout tier
- **ADR-048**: Herald invocation bridge — CLI `npx thevoidforge herald --command /<name> --json`
- Command files reference "Silver Surfer Pre-Scan (ADR-048)" instead of "Herald Pre-Scan (ADR-047)"
- Command-doc sync fixes from /assess audit (v23.5.4)

### Audit triage — what's already fixed vs outstanding:

**Already fixed in v2.8.0 (18 findings):** Retired flag promotion (5), /build→/campaign primary (2), 14→13 phase count (3), /commands/build agent assignment, --autonomous marking, --continuous verification, /git --dry-run, deploy targets, historical claims (3).

**Outstanding (16 items):**

2. Silver Surfer agent data + tier breakdown update (38→39 scouts)
3. About page test count: "1,340+" → "1,384"

## Decision

### 1. Upstream-first, then content fixes

Sync methodology files first (v23.5.4 + v23.6.0) so that agent definitions, command files, and method docs are current before updating site content. This ensures Silver Surfer's agent definition exists in `.claude/agents/` before we reference it in site data.

### 2. Add Silver Surfer to agents.ts as scout tier

Silver Surfer (Norrin Radd) joins the Marvel universe as a Haiku-tier scout. Agent #264. stats.ts auto-computes — no hardcoded count updates needed in components that use `display.*` or `stats.*`.

### 3. Hardcoded tier breakdown: update per ADR-020 precedent

The agents page tier grid (20/190/38/15) stays hardcoded per ADR-020. Update scouts from 38 → 39. Total display via `stats.totalAgents` auto-updates to 264.

### 4. Agents page hero: show total agents, not just sub-agents

Change the PageHeader subtitle from `${subAgents.length}+ sub-agents` to `${stats.totalAgents}+ agents` for consistency with homepage messaging. Keep the tier breakdown grid below for the detailed split.

### 5. Historical release narratives: document, don't compute

Release notes in releases.ts contain counts that were accurate at release time (e.g., "150+ named sub-agents" for v1.0.0). These are historical records. Replacing them with computed current values would be anachronistic. Add a code comment documenting this pattern. Fix only the one release note that was wrong at time of writing: v20.1.0 "40 total" patterns → "37 total".

### 6. Speech bubble animation: ScrollReveal wrapper approach

Add animations by wrapping SpeechBubble instances in ScrollReveal on each page. This:
- Keeps SpeechBubble as a server component (no bundle impact)
- Reuses the existing ScrollReveal pattern (consistency)
- Follows ADR-018 (above-fold first section: no animation)
- Respects reduced-motion automatically

Per ADR-018, the first SpeechBubble on each page (typically above-fold) gets NO animation. Subsequent bubbles get ScrollReveal with staggered delays.

### 7. Herald + --focus as homepage feature

Add a feature card or hero callout for Herald intelligent dispatch. This is a major v23.5 capability that differentiates VoidForge. Position it as "automatic agent team selection" — Haiku pre-scans code, selects optimal agents, no manual configuration.

### 8. Stats.ts expansion

Add computed values for method docs count and ADR count. These are marketing-worthy numbers (29 method docs, 48 ADRs) that demonstrate methodology depth.

## Consequences

### Enables:
- Site accuracy against v23.6.0 source of truth
- Silver Surfer as the 264th agent across marketing materials
- Herald dispatch as a prominent feature
- Consistent animation language across all speech bubbles
- Campaign state reflects actual history

### Prevents:
- Users seeing stale retired flags or wrong counts
- Confusion between sub-agent count and total agent count on /agents
- Speech bubbles feeling static compared to animated sections around them

### Trade-offs:
- Wrapping ~30 speech bubbles in ScrollReveal touches many files but is mechanical
- Some prophecy page counts may be hardcoded in release narratives — we accept these as historical snapshots rather than computing them dynamically (complexity not justified for rarely-changing archival text)

## Alternatives Considered

1. **AnimatedSpeechBubble wrapper component** — Creates a new client component that wraps SpeechBubble + ScrollReveal. Rejected: adds a component for what's essentially `<ScrollReveal><SpeechBubble/></ScrollReveal>` — premature abstraction per coding standards.

2. **Convert SpeechBubble to client component with built-in motion** — Rejected: moves a pure server component to client-side, adds to bundle for every page using it, and couples animation logic into the presentation component.

3. **Compute all release narrative counts dynamically** — Rejected: historical release notes should reflect what was true at release time. Computing current values in historical contexts is misleading.

## Implementation Scope

Fully implemented in v2.9.0 via campaign execution. No stub code. All changes are content updates, data additions, and ScrollReveal wrapping — no new infrastructure.
